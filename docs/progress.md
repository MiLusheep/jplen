### 2026-05-19 v0.6.0 新闻阅读系统 + GitHub Pages部署
- 新闻自动抓取系统
  - 创建多源日语新闻抓取脚本（scripts/fetch-news.mjs）
  - 支持数据源：NHK Easier（RSS，含完整振假名标注）、NHK速報（RSS）、Yahoo!ニュース（RSS）
  - NHK Easy News Web Easy需要Cookie认证，当前环境下认证流程不稳定，暂用NHK Easier替代
  - 新闻内容自动难度分级算法（基于汉字比例和文本长度，N1-N5）
  - 数据去重（基于标题相似度）和质量筛选（内容过短的文章过滤）
  - 抓取结果保存到 src/data/news/articles.json
  - 当前测试：成功抓取38篇文章（NHK Easier 15篇 + Yahoo 23篇）
- GitHub Actions 自动化
  - 创建 .github/workflows/fetch-news.yml
  - 定时任务：每日15:00 UTC（日本时间00:00）自动执行
  - 支持手动触发（workflow_dispatch）
  - 抓取后自动检测数据变更，有变更则提交并触发重新部署
- 前端新闻阅读集成
  - 创建新闻服务模块（src/services/newsService.ts），带本地缓存
  - 阅读页面新增「新闻热点」标签页，与「教材阅读」并列
  - 新闻支持按级别（N5-N1）和来源筛选
  - 新闻卡片展示：来源标签、级别标签、标题、摘要、字数、阅读时间
  - 新闻详情页：展示带振假名（ruby标签）的原文内容，支持查看原文链接
  - NHK Easier的文章包含完整ruby标注，非常适合日语学习者
- GitHub Pages 部署
  - 项目部署到 GitHub Pages
  - 路由改用 HashRouter 适配静态托管
  - Vite 配置 base 路径
  - .gitignore 排除 .trae 文件夹
